ElasticSearch 入门

Part1.基本概念：索引,文档和Rest Api

索引和文档偏向于开发人员的一个角度，属于一个逻辑上的概念。

一、名词解释

文档 : ElasticSearch 中可以被搜索数据的最小单位，类似关系型数据库中的行。

  • 文档会被序列化成Json格式存储在ElasticSearch中
  • 每个文档都会有一个唯一的ID

文档的元数据 ：用于标注文档的相关信息

  • _index ： 文档所属的索引名
  • _type : 文档所属的类型
  • _id : 文档唯一Id
  • _source : 文档的原始Json数据
  • _all ： 已被废除，所有字段
  • _version ：文档的版本信息
  • _score ： 相关性打分

索引(_index) ：索引是文档的容器，是一类分档的结合

  • Mapping ：定义文档字段的类型
  • Setting ： 定义不同的数据分布

type : 7.0 版本之后一个索引只会对应一个type

锁 Lock接口

对于使用者来说Lock接口提供了如下的方法
  • void lock() 获取锁
  • void lockInterruptibly() 可以中断的获取锁
  • boolean tryLock(). 尝试非阻塞的获取锁
  • void unLock() 释放锁
  • Condition newCondition() 获取等待通知组件
队列同步器
关于同步状态相关的方法
  • getState()
  • setState()
  • compareAndSetState()
同步队列器可以重写的方法
  • tryAcquire 和 tryAcquireShared 获取同步状态
  • tryRelease 和 tryReleaseShared 释放同步状态
  • isHeldExclusively(). 判断是否是独占模式
同步队列器实现

靠双向队列来完成同步状态的管理，每一个Node代表一个线程。
同步队列器独占锁 && 共享锁 。独占和共享锁 又分为 公平或者非公平。独占锁就是ReentrantLock。共享锁是读写锁的读锁

  • Node的状态有5种

      1. Cancelled
      1. SINGAL 后续节点处于等待的状态
      1. CONDITION 处于等待队列中
      1. PROPAGATE
      1. INTIAL 初始状态

Mockito

Mockito是什么

是一个强大的用于 Java 开发的模拟测试框架, 通过 Mockito 我们可以创建和配置 Mock 对象, 进而简化有外部依赖的类的测试.

  • 英文文档：https://static.javadoc.io/org.mockito/mockito-core/3.0.0/org/mockito/Mockito.html
  • 中文文档：https://github.com/hehonghui/mockito-doc-zh/blob/master/README.md#0

使用 Mockito 的大致流程如下:

  • 创建外部依赖的 Mock 对象, 然后将此 Mock 对象注入到测试类中.
  • 根据业务逻辑进行mock设置.（打桩）
  • 把需要Mock的对象进行替换后执行测试代码.
